# Product Workflows

This document describes the key workflows and user journeys in the Carrier Ops Hub.

## Dispatcher Workflows

### Creating a New Load

1. Navigate to Dispatch Dashboard
2. Click "New Load" button
3. Enter load details:
   - Pickup location and date/time
   - Delivery location and date/time
   - Rate and customer information
4. Assign driver and vehicle (optional at this stage)
5. Save load

### Assigning a Load to a Driver

1. View unassigned loads in Dispatch Dashboard
2. Click on a load
3. Select "Assign Driver"
4. Choose available driver from list
5. Select vehicle
6. Confirm assignment

### Monitoring In-Transit Loads

1. View "In Transit" section of dashboard
2. Real-time location updates from Samsara/Motive
3. Monitor ETA and potential delays
4. Review event history
5. Communicate with driver via messaging

## Driver Workflows

### Viewing Assigned Loads

1. Log in to driver portal
2. See "My Loads" on home screen
3. View details of upcoming and current loads

### Updating Load Status

1. Arrive at pickup/delivery location
2. Click "Arrived at Pickup" or "Arrived at Delivery"
3. System records timestamp and location
4. Upload required documents (BOL, POD)
5. Mark stop as complete

### Submitting Expenses

1. Take photo of receipt
2. Navigate to load
3. Click "Add Expense"
4. Select expense type and enter amount
5. Attach receipt photo
6. Submit for approval

## Billing Workflows

### Processing Completed Loads

1. View "Ready for Billing" queue
2. Verify all documents are present (BOL, POD, Rate Confirmation)
3. Create invoice
4. Send to customer
5. Track payment status

## Safety Workflows

### Reviewing Driver Compliance

1. Access Safety Dashboard
2. View HOS (Hours of Service) compliance
3. Review driver inspection reports
4. Monitor violations or alerts
5. Take corrective actions as needed

## Common Workflows

### Document Management

1. Upload documents to load
2. Categorize by type (BOL, POD, etc.)
3. View/download documents
4. Share with relevant parties

### Event Timeline

1. View chronological event history for a load
2. System-generated events (status changes, location updates)
3. User-generated events (notes, communications)
